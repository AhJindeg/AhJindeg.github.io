Skill 模板设计

设计可扩展、可审查、可维护的 Skill Template 结构。

1) Skill Template 的定位

Skill Template 是“可执行接口”，不是“经验笔记”。
它要把能力边界、输入输出契约和失败处理讲清楚，保证不同执行者得到一致结果。

2) 设计目标

  • 快速上手：新成员可直接调用；
  • 边界清晰：适用/不适用场景明确；
  • 可审查：执行步骤可追溯；
  • 可演进：版本升级不破坏调用方。

3) 建议字段（推荐最小集）

  • 场景定义：目标、前提、约束；
  • 输入要求：必填字段、可选字段、默认值；
  • 执行步骤：可按序执行的动作；
  • 输出规范：格式、字段、错误码；
  • 失败处理：重试策略、降级策略、人工介入条件；
  • 安全边界：权限限制、敏感操作白名单/黑名单；
  • 版本信息：兼容性说明与变更记录。

4) 演进策略

  1. 语义化版本：major.minor.patch；
  2. 向后兼容优先：新增字段优先，不随意改语义；
  3. 灰度发布：先小流量验证，再全量替换；
  4. 弃用机制：标记 deprecated 并提供迁移路径。

5) 反模式

  • 模板只写“做什么”，不写“怎么判定完成”；
  • 输出无结构，无法接入自动化流程；
  • 没有错误策略，失败后只能人工兜底。
