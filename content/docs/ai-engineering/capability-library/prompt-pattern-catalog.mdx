---
title: Prompt 模式目录
description: 按任务类型组织高复用 Prompt 模式并定义适用边界。
---

## 1) 为什么要目录化

Prompt 资产化的核心价值是“可复用”和“可治理”。  
如果只靠个人记忆，同样任务会反复重写、质量波动很大。

## 2) 推荐分类维度

- 任务类型：信息抽取、结构化生成、代码修改、审查对比；
- 风险等级：低风险自动化 / 中风险复核 / 高风险人工审批；
- 输出契约：文本、JSON、Patch、Checklist 等；
- 失败特征：幻觉、遗漏、格式错误、越权动作。

## 3) 每个模式必须包含的字段

- 场景定义：适用于什么问题，不适用于什么问题；
- 输入前提：必须提供哪些上下文；
- 执行指令：步骤和约束要可执行；
- 输出规范：格式、长度、字段、禁止项；
- 失败信号：什么现象代表这次结果不可用；
- 样例：最小输入样例与标准输出样例。

## 4) 质量治理建议

1. 建立版本号与变更日志（为什么改、改了什么）；
2. 对高频模式做 A/B 对比，保留胜出版本；
3. 定期清理低命中、低收益模式，避免目录膨胀；
4. 将“失败案例”纳入目录，形成负面样本库。

## 5) 反模式

- 只收藏“通用万能 Prompt”；
- 没有输出契约，导致结果不可自动消费；
- 只写成功案例，不记录失败条件与禁用边界。
