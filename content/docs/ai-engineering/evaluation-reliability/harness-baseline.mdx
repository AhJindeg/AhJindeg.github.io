---
title: Harness 基线设计
description: 构建可重复执行的 Harness Baseline 以衡量能力变化。
---

## 1) Harness 解决什么问题

Harness 用于回答一个核心问题：  
这次改动是“真实提升”，还是“样例碰巧更好”。

## 2) 评测集设计原则

- 覆盖面：高频场景 + 高风险场景 + 长尾场景；
- 分层：冒烟集（快）、回归集（稳）、完整集（全）；
- 可追溯：每条样本有来源、标签、评分依据；
- 可演进：定期剔除过期样本并补充新故障样本。

## 3) 指标体系建议

- 正确性：任务完成率、结构化输出合法率；
- 可靠性：失败率、重试成功率、结果一致性；
- 经济性：平均成本、P95 成本、缓存命中收益；
- 性能：P50/P95 时延、超时率、队列堆积。

## 4) 固定实验条件

为保证可比性，至少固定以下变量：

- 模型版本与关键参数；
- 检索索引快照与配置；
- Prompt 模板版本；
- 工具调用策略与超时阈值。

## 5) 输出物建议

每次评测都应产出：

- 汇总报告（核心指标变化）；
- 差异样本列表（变好/变差案例）；
- 结论建议（可发布 / 需复核 / 阻断发布）。

## 6) 常见反模式

- 只保留“最终分数”，不保留样本证据；
- 每次改动都换评测集，无法横向比较；
- 不区分失败严重度，导致高风险问题被平均分掩盖。
