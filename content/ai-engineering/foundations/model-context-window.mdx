## 为什么重要

Context Window 不只是 token 上限，更是稳定性边界。  
当 Prompt 超过这个边界时，通常先出现质量下降，再出现显式失败。

## 实践规则

- 把稳定指令放在前面，把变化信息放在后面。
- 一个 Prompt 尽量只服务一个目标。
- 历史对话优先总结，不要整段回放。
- 长期知识放到外部存储，按需检索注入。

## 常见失败模式

将需求、日志、旧尝试混在一个超长 Prompt 里，往往会产生冲突输出。  
更稳妥的做法是先拆任务，再分阶段执行。
